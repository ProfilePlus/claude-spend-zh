Claude Code token 工作原理

每次你发送一条消息，Claude 不仅仅读取你最新的一条消息。它会从头开始重新读取整个对话。这就是为什么 token 消耗得很快。

示例：一个 3 条消息的对话
第 1 条 你发送"帮我修复这个 bug"。Claude 全新读取你的消息（输入）并保存对话到缓存（缓存写入）。 约 4K token
第 2 条 你回复"试试另一个文件"。Claude 从缓存中复用第 1 条（缓存读取 — 便宜 10 倍），只全新处理你的新消息（输入）。然后保存更新后的对话（缓存写入）。 约 8K token
第 3 条 Claude 从缓存复用第 1-2 条（缓存读取）并全新处理你的新消息（输入）。Claude 的回复是输出。 约 14K token
如果没有缓存，每条消息都要按原价处理。缓存读取可以帮你节省最多 10 倍的重复上下文费用。
输入 — 你的消息 + 对话历史，全新处理。按原价计费。
缓存写入 — Claude 保存你的对话，以便下次不必重新处理所有内容。费用是输入的 1.25 倍，但只发生一次。
缓存读取 — 后续消息复用已保存的上下文。比输入便宜 10 倍。缓存读取越多 = 节省越多。
输出 — Claude 写回的内容。每个 token 单价最高，但通常数量最少。
提示：切换任务时使用 /clear。这会重置对话，让 Claude 停止重新读取你不再需要的旧上下文。

此对话使用的 token 总数。越高费用越高。长对话成本更高，因为 Claude 每轮都要重新读取所有内容。
 读取
Claude 读取的 token：你的消息、对话历史、文件内容和系统上下文。这会随着每条消息递增，因为完整历史会被重新发送。
 写入
Claude 写回的 token：回复、代码、解释。通常只占总费用的一小部分。
